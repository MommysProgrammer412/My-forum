# MVP - Базовый функционал

## По ролям
## 1. Польлзователь

**Регистрация**
- При регистрации указать: имя пользователя, пол, дата рождения (без года), почта, пароль, пароль еще раз.
- При регистрации нажать: зарегистрироваться.

**Вход**
- При входе указать: почту, пароль.
- При входе нажать: войти, забыл пароль.

**Восстановление пароля**
- При восстановлении пароля указать: почту.
- При восстановлении пароля нажать: получит письмо.
- Уведомление: `на вашу почту было отправлено письмо с сылкой для восстановления пароля`.

**Разделы**
- Профиль, главная, поиск, уведомления, реакции, настройки.
**Профиль**
- Отображается: аватарка, имя пользователя, статус, количество постов/подписчиков/подписок, список постов/ретвитов пользователя.
- Возможно: создать пост, взаимодействие с постами, прокрутка вниз, заблокировать пользователя.
**Главная (открывается по умолчанию при входе)**
- Отображается: посты подписок.
- Возможно: взаимодействие с постами, прокрутка вниз.
**Поиск**
- Отображается: поисковая строка.
- Возможно: поиск профиля и постов с хэштэгом.
**Уведомления**
- Отображается: ретвит/лайки/комментарии поста пользователя/новые подписчики/день рождения подписок.
**Реакции**
- Отображается: лайкнутые посты и комментарии.
- Возможно: взаимодействие с постами/комментариями, прокрутка вниз.
**Настройки**
- Отображается: информация о профиле пользователя, черный список.
- Возможно: выйти/удалить аккаунт, изменить почту/пароль, изменить/удалить аватарку, изменить имя пользователя, изменить статус, разблокировать пользователей из черного списка.
**Посты**
- При создании возможно: добавлять хэштэги/вложение/текст поста.
- После создания можно: писать комментарии под ним, удалить/редактировать свой пост, поставить/снять лайк, ретвитнуть в свой профиль (с возможностью дополнить ретвит информацией как обычный пост), закрепить пост/ретвит в своем профиле, подать жалобу.
**Комментарии**
- Возможно: писать/редактировать/удалять свои комментарии, поставить/снять лайк, подать жалобу.

## 2. Администратор

**Имеет весь функционал пользователя и возможность**
- Заблокировать/разблокировать: пользователя, пост, комментарий, хэштэг и посты с этим хэштэгом в разделе `жалобы` или при ручном поиске.
- В разделе `скрытое` посмотреть и разблокировать нежелательный контент.

>## MVP
>Если вы сразу попытаетесь построить «космический корабль», вы никогда его не закончите. Программисты строят «самокат», потом «велосипед», потом «мотоцикл» и только в конце — «машину».
>
>Вот как выглядит этот цикл в реальности 2026 года:
>1. Этап «Ядро» (MVP)
>Программист реализует только критическую цепь событий.
>
>    Пример: Вы делаете сайт для заметок. База — это текстовое поле и кнопка «Сохранить». Всё. Без этого проекта нет.
>    Результат: У вас есть работающий код, который можно потрогать. Это дает дофамин и мотивацию продолжать.
>
>2. Этап «Наращивание мяса» (Backlog)
>У разработчика есть список Backlog (бэклог) — это корзина идей, которые он сознательно отложил. Когда «ядро» работает, он достает оттуда функции по приоритету:
>
>    «Так, заметки сохраняются. Теперь добавлю возможность их удалять».
>    «Удаление работает. Теперь добавлю поиск (Search Bar)».
>
>3. Этап «Рефакторинг» (Очистка)
>Это важный момент, который новички часто упускают. Когда вы добавляете новую фичу, старый код может начать «мешаться». Программист останавливается и переписывает часть старого кода так, чтобы он стал гибче и смог принять будущие обновления.
>Почему это правильно с точки зрения обучения?
>
>    Снижение когнитивной нагрузки: Ваш мозг фокусируется на одной конкретной проблеме (например, как работает поиск), а не на всей сложности сайта сразу.
>    Архитектурное мышление: Вы учитесь писать код так, чтобы его можно было расширить. Если вы написали код «намертво», вы не сможете добавить второстепенную фичу, не сломав всё остальное. Это и есть главный урок программирования.
