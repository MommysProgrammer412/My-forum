# Порядок действий для views (только логика):

## 1. Аутентификация (регистрация, вход)

`Что делает view?` Пока не пиши вручную. Django предоставляет готовые классы (LoginView, LogoutView). Просто импортируй их в users/views.py и используй в urls.py. Для регистрации (SignUp) можно создать свою view по аналогии с create_post_view.

## 2. Создание поста

`Что делает view?` Два сценария:
- GET запрос: Показывает пустую форму для создания поста.
- POST запрос: Принимает данные из формы, создает новый объект Post (и связанные File, если есть вложения), сохраняет в базу, перенаправляет обратно на ленту.

`Какие данные нужны?` Для POST — данные формы (текст, файлы). Важно проверить, что автором поста был установлен текущий пользователь (request.user).
`Какой шаблон использовать?` Пока можешь не создавать отдельный шаблон. Можно добавить форму прямо в index.html.
`Название view:` create_post_view в posts/views.py.

## 3. Добавление комментария

`Что делает view?` Принимает POST-запрос с текстом комментария и id поста. Создает новый объект Comment, устанавливает author как request.user и post по id, сохраняет. Затем перенаправляет на страницу этого поста (post_detail).
`Какие данные нужны?` id поста, текст комментария.
`Какой шаблон использовать?` Не используется напрямую, но форма для комментария будет в post_detail.html.
`Название view:` add_comment_view в posts/views.py.

## 4. Профиль пользователя (пока только его посты)

`Что делает view?` По username (или id) находит объект User и все его посты (Post.objects.filter(author=...)).
`Какие данные нужны?` Объект User и список его постов.
`Какой шаблон использовать?` У тебя есть my_posts.html. Эта view передаст туда пользователя и его посты.
`Название view:` user_posts_view в users/views.py.

## 5. Просмотр одного поста (и комментарии)

`Что делает view?` По id поста находит один объект Post и все его комментарии (Comment.objects.filter(post=...)), сортируя их по дате.
`Какие данные нужны?` Один объект Post и список объектов Comment.
`Какой шаблон использовать?` Пока нет, но логично создать post_detail.html. View передаст туда пост и комментарии.
`Название view:` post_detail_view в posts/views.py.

## 6. Начни с главной страницы (лента постов)

`Что делает view?` Запрашивает из базы все посты от авторов, на которых подписан текущий пользователь (и его собственные), сортирует их по дате (свежие сверху).
`Какие данные нужны?` Список объектов Post. Помни, что у Post есть author, text, created_at, attachment и т.д.
`Какой шаблон использовать?` Похоже, что у тебя уже есть index.html. Эта view передаст список постов в index.html.
`Название view:` Например, feed_view или index_view в posts/views.py.

## Ключевые моменты для тебя прямо сейчас (view в posts):

- Открой posts/views.py. Сейчас там только импорт render.
- Напиши одну view за раз. Начни с первой: ленты постов (feed_view).
- Логика функции:
    Импортируй Post из posts.models.
    posts = Post.objects.all().order_by('-created_at') — пока просто все посты (позже усложнишь фильтрацию по подпискам).
    return render(request, 'index.html', {'posts': posts}) — передаём список постов в шаблон.

- Затем создай URL в urls.py, чтобы этот view стал доступен по адресу, например, /.
